## Step 1: Add GitHub as Remote

If you haven't added GitHub as a remote yet, run:

```bash
git remote add github https://github.com/Mastrobono/lite-tech-challenge.git
```

## Step 2: Verify Remotes

```bash
git remote -v
```

You should see something like:
```
github  https://github.com/Mastrobono/lite-tech-challenge.git (fetch)
github  https://github.com/Mastrobono/lite-tech-challenge.git (push)
origin  https://gitlab.com/... (fetch)
origin  https://gitlab.com/... (push)
```

## Step 3: Sync Options

### Option A: Automatic Script (Recommended)

**On Windows (PowerShell):**
```powershell
.\scripts\sync-remotes.ps1
```

**On Linux/Mac:**
```bash
chmod +x scripts/sync-remotes.sh
./scripts/sync-remotes.sh
```

### Option B: Manual Push to Both

```bash
# Push to GitLab
git push origin feat/initial-setup

# Push to GitHub
git push github feat/initial-setup
```

### Option C: Create a Git Alias

```bash
# Create alias to push to both
git config alias.pushall '!git push origin && git push github'

# Use the alias
git pushall
```

### Option D: Configure Multiple Push URLs (More Automatic)

You can configure Git to automatically push to both remotes:

```bash
# Add GitHub as additional push URL to origin
git remote set-url --add --push origin https://gitlab.com/...
git remote set-url --add --push origin https://github.com/Mastrobono/lite-tech-challenge.git

# Now git push origin will push to both
git push origin feat/initial-setup
```

**Note:** With this option, `git push origin` will push to both, but `git push` (without specifying) will only push to the first one.

## Recommendation

For better control, I recommend using the **script** (Option A) or the **alias** (Option C), because:
- âœ… You have explicit control over when to sync
- âœ… You can see the results of each push
- âœ… It's easier to debug if something fails

## Troubleshooting

### Error: "remote github already exists"
```bash
# View current remotes
git remote -v

# If github already exists but points to another repo, update it:
git remote set-url github https://github.com/Mastrobono/lite-tech-challenge.git
```

### Error: "Authentication failed"
Make sure you have your credentials configured:
- For GitHub: You can use Personal Access Token or SSH
- For GitLab: Verify your credentials

### Verify that both remotes are configured
```bash
git remote show origin
git remote show github
```
